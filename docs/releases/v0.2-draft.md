# VibeLog v0.2 Draft

## Summary

VibeLog v0.2 draft upgrades the project from a Markdown-first logging skill into a hook-friendly process record standard for vibe-built products.

The main shift:

```txt
v0.1: record the vibe process manually
v0.2: prepare the vibe process to be recorded automatically by agent hooks
```

## Core Model

```txt
Vibe Repo = VibeLog + Project Artifacts
VibeLog = process memory
Project Artifacts = product evidence
```

## What Changed

- Added `creation_mode` so VibeLog can describe human/AI participation without judging it.
- Added `process_level` with `minimal`, `core`, and `full` modes.
- Added `Validation Design` as a first-class section.
- Added `Verification Evidence` as a first-class section.
- Added `Artifact Index` for product evidence references such as code, demos, screenshots, test outputs, releases, and agent configs.
- Added `Handoff State` so another agent can continue the project safely.
- Added `Public / Private Projection` and `Branch / Remix Readiness` for future VibeHub use.
- Added Claude Code hook adapter notes.
- Updated README, skill metadata, template, schema, format reference, and self-recorded project VibeLog.

## Automation Direction

The first adapter target is Claude Code.

Recommended hook mapping:

```txt
SessionStart      -> load concise VibeLog context
UserPromptSubmit  -> classify prompt and record engineering prompts
PostToolUse       -> capture edits, commands, tests, artifacts, and verification evidence
Stop              -> update progress, implementation status, handoff, and JSON export
PreCompact        -> preserve essential context
PostCompact       -> refresh handoff state if useful
```

## Compatibility

The JSON schema accepts both:

```txt
vibelog@0.1
vibelog@0.2-draft
```

Existing v0.1 logs can be reconstructed or upgraded gradually.

## Not Included Yet

- Deterministic Markdown-to-JSON exporter.
- Claude Code hook implementation.
- Codex hook implementation.
- Public VibeHub website.
- Marketplace, payments, or public collaboration features.

## Next Step

Build a minimal Claude Code hook adapter that can update `vibe-log.md` and regenerate `vibe-log.json` during normal vibe coding.
