# BillMate Lite Dogfood Example

This example shows a VibeLog generated from an agent-simulated scratch project.

The scratch project was a tiny local bill splitter used to test whether VibeLog can record:

- idea intake and evolution
- human-in-the-loop direction
- engineering execution prompts
- test-first implementation
- development logs
- bugfix / incident thinking
- validation design
- verification evidence
- handoff state

This directory intentionally includes only generated VibeLog output:

- `vibe-log.md`
- `vibe-log.json`

It does not include the scratch project source code. The source remained local at the time of testing and is referenced only as private artifact context inside the generated log.
