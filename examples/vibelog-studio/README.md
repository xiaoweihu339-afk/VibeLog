# VibeLog Studio Dogfood Example

This example shows a VibeLog generated while using the `vibelog` skill during a real engineering session.

It intentionally includes only:

- `vibe-log.md`: the human-readable process record
- `vibe-log.json`: the structured export

It does not include the VibeLog Studio application source code. The purpose of this example is to demonstrate how VibeLog records a vibe coding process: idea evolution, human-in-the-loop decisions, execution prompts, implementation status, development logs, bug fixes, verification evidence, and handoff state.

Use this example as a reference for what a completed local VibeLog can look like after an agent-assisted project session.
